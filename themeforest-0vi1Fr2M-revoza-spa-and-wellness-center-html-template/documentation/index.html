License FAQ

If I need to use the template on a second site, do I need a second license?

Yes. One template purchase allows you to use it on one finished site. If you need to use it on a second site, you must purchase a second license.

Can I use the template on a test site and then move it to a live site when finished?

Yes, you can. You can keep the template on a test site to build it, then move it to the live site once you’re finished.

Can I keep a test and live sites with one Regular License purchase?

You can keep one test/staging website and one live website with one Regular License. Keep in mind that you must use the same domain.

System Requirements

Before installing the template, please make sure your system meets the following requirements:.

  • A modern web browser (Chrome, Firefox, Edge, Safari)
  • A text or code editor (VS Code, Sublime Text, Notepad++)
  • Web hosting or local server
  • PHP 7.4 or higher (required for contact form functionality)

Package Contents

After downloading and unzipping the template package, you will find the following folder structure inside html folder:

  • css - Stylesheets Folder
  • webfonts - Icon Font Folder
  • images - Demo Images Folder
  • js - Javascripts Folder
  • form-process.php - Contact Form Process File
  • html/*.html - All the templates

Installation

  1. Local Installation (for testing)
    • Install XAMPP or WAMP on your computer.
    • Copy all files from the html folder.
    • Paste them into the htdocs (XAMPP) or www (WAMP) directory.
    • Open your browser and visit: http://localhost/your-folder-name/
  2. Live Server Installation
    • Log in to your hosting control panel (cPanel).
    • Open File Manager and navigate to public_html.
    • Upload all files from the html folder.
    • Open your domain in the browser to verify installation.

Important: Contact form will not work on file:// protocol. A server is required.

Color Customization

You can change your Website’s Color Scheme in an instant. You simply need to set the your Color Code in the css/custom.css file using the following code:

Code Example

								:root{
	--primary-color		: #000000;
	--secondary-color	: #EFF2E7;
	--bg-color		: #FFFEF6;
	--text-color		: #666666;
	--accent-color		: #585E48;
	--white-color		: #FFFFFF;
	--divider-color		: #0000001A;
	--dark-divider-color	: #FFFFFF1A;
	--error-color		: rgb(230, 87, 87);
}

Changing Fonts

By default, Revoza uses Fonts namely: Bricolage Grotesque from the Google Fonts Library You can change the fonts from css/custom.css file in below lines..

Code Example

								:root{
	--default-font	: "Bricolage Grotesque", sans-serif;
}

Adding New Fonts

  • Using Google Fonts

    
<link href="https://fonts.googleapis.com/css2?family=Bricolage+Grotesque:opsz,wght@12..96,200..800&display=swap" rel="stylesheet">

    In order to change the Fonts, you will need to Edit the Above Links with your Chosen Google Fonts.

  • Using Self-Hosted Fonts

    
@font-face {
  font-family: 'MyWebFont';
  src: url('webfont.woff2') format('woff2'), /* Super Modern Browsers */
       url('webfont.woff') format('woff'), /* Pretty Modern Browsers */
       url('webfont.ttf')  format('truetype'), /* Safari, Android, iOS */
       url('webfont.svg#svgFontName') format('svg'); /* Legacy iOS */
}

    More Examples for using Self Hosted Fonts.
